### Not-yet-implemented cases (📋)

These test cases are identified and ready to implement once the environment supports them:

- **`plan --base-commit`**: Requires a known commit SHA with a changed file. Could be implemented by creating a temp commit in the test.
- **`plan` increment modes with existing tags**: Needs existing version tags in the repo. Could be implemented against a known app that has been tagged.
- **`changes --base-commit HEAD~1`**: Requires known-changed files. Can be implemented with a temp file change + commit.
- **`release-notes --previous-tag`**: Needs an existing tag to compare against.
- **`create-combined-github-release-with-notes` no version**: Pure validation, no API call needed — implement in `test_cli_integration.sh`.
- **`release-multiarch --dry-run`**: Needs Bazel for app lookup; implement in `test_cli_integration_bazel.sh`.
- **`plan-helm-release --charts`**: Needs a known chart name; query with `list-helm-charts` first.
- **`unpublish-helm-chart` with real index fixture**: Add a test fixture `index.yaml` and test round-trip.
